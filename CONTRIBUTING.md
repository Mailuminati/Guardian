# Contributing to Mailuminati Guardian

Thank you for your interest in contributing to Mailuminati Guardian.

Guardian is part of the Mailuminati ecosystem and focuses on local first,
privacy preserving, and high performance email threat detection.

We welcome contributions from operators, developers, researchers,
and anyone running email services.

---

## Guiding Principles

When contributing, please keep the following principles in mind:

- Local first detection and enforcement
- Privacy by design, no raw email content sharing
- Performance matters, Guardian is designed for high volume environments
- Simplicity over cleverness
- Operational realism over theoretical purity

---

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Proposing improvements or new features
- Improving documentation
- Reviewing code or design proposals
- Sharing operational feedback from real deployments

---

## Reporting Bugs

Please include as much information as possible:

- Guardian version
- Operating system and architecture
- How Guardian is deployed
- Steps to reproduce the issue
- Expected vs actual behavior

If the issue may have security implications, please follow the
security reporting process described in `SECURITY.md`.

---

## Pull Requests

Before submitting a pull request:

- Ensure the code builds cleanly
- Keep changes focused and scoped
- Avoid unrelated refactoring
- Document any new behavior or configuration

Pull requests should include:

- A clear description of the change
- Rationale for the approach
- Any known limitations or trade offs

---

## Commit Style

There is no strict commit message format, but commits should:

- Be descriptive
- Be logically scoped
- Avoid mixing unrelated changes
